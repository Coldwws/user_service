# 🎯 Указатель vs Значение в DI контейнере - ПРАВИЛО

## ☝️ ОДО ПРАВИЛО (легко запомнить):

```
Указатель (*) нужен ЕСЛИ:
1. Это РЕСУРС (БД, сеть, файлы) 
2. ИЛИ это большой объект (экономия памяти)
3. ИЛИ нужно ИЗМЕНЯТЬ состояние

БЕЗ указателя ЕСЛИ:
- Это КОНФИГ (только читаем)
- Это СЕРВИС/ИНТЕРФЕЙС (логика, не состояние)
- Это СТРУКТУРА-ОБРАБОТЧИК (вызываем методы)
```

---

## 📋 Таблица для вашего проекта:

| Компонент | Тип | Указатель? | Почему |
|-----------|-----|-----------|--------|
| `Config` | конфиг | ❌ | Only read (читаем, не меняем) |
| `pgPool` | ресурс БД | ✅ | Resource (управляет соединениями) |
| `gRPC Server` | ресурс сети | ✅ | Resource (слушает сеть) |
| `UserRepository` | interface | ❌ | Это interface, не нужен указатель |
| `UserService` | interface | ❌ | Это interface, не нужен указатель |
| `UserServer` | struct | ❌ | Only methods (только методы) |
| `Logger` | interface | ❌ | Это interface |
| `Database` | объект | ✅ | Resource (управляет подключениями) |

---

## 💡 ПРИМЕРЫ:

### ✅ УКАЗАТЕЛЬ нужен:

```go
pgPool *pgxpool.Pool        // Ресурс - управляет памятью, соединениями
grpcServer *grpc.Server     // Ресурс - слушает порт, имеет состояние
logger *slog.Logger         // Может писать в файл (ресурс)
cache *redis.Client         // Ресурс - сетевое подключение
file *os.File               // Ресурс - открытый файл
```

### ❌ УКАЗАТЕЛЬ НЕ нужен:

```go
cfg config.Config           // Конфиг - только читаем
service UserService         // Интерфейс - это не объект
repository UserRepository   // Интерфейс - это не объект
handler UserHandler         // Структура-обработчик - только методы
converter Converter         // Структура-конвертер - только методы
validator Validator         // Интерфейс или структура - только логика
```

---

## 🔍 КАК ПРОВЕРИТЬ себя:

**Вопрос себе:**
1. "Это управляет ресурсом (БД, файл, сеть)?" → **ДА** = `*pointer`
2. "Это просто интерфейс?" → **ДА** = `interface`
3. "Это большой объект (>1KB)?" → **ДА** = `*pointer`
4. "Состояние будет меняться?" → **ДА** = `*pointer`
5. Всё остальное = `значение`

---

## 📝 ШПАРГАЛКА по GO:

```go
// В Go интерфейсы уже содержат указатель
type UserService interface {
    Get(id int) User
}

var svc UserService  // ← НЕ пишем *UserService!
                     // интерфейс сам хранит указатель

// Конкрет структура - может быть по значению
type MyService struct {
    field string
}

var s MyService      // ✅ по значению
var s *MyService     // ✅ по указателю (если нужно менять)
```

---

## 🎓 ВЫВОД:

**В вашем DI контейнере:**

- `pgPool *pgxpool.Pool` ← указатель (РЕСУРС)
- `grpc *grpc.Server` ← указатель (РЕСУРС)
- `userRepository UserRepository` ← БЕЗ (интерфейс)
- `userService UserService` ← БЕЗ (интерфейс)
- `userServer UserServer` ← БЕЗ (структура-обработчик)

---

**Правило просто: Ресурсы = указатель, остальное = значение** 👍
